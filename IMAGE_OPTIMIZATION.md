# מדריך אופטימיזציה של תמונות

## הבעיה
התמונות שלך גדולות (300-360KB כל אחת), מה שגורם לטעינה איטית ב-GitHub Pages.

## פתרונות

### 1. דחיסת תמונות (מומלץ ביותר)

#### אופציה A: שימוש בכלי אונליין
1. לך ל-https://squoosh.app/
2. העלה את התמונות מ-`public/images/`
3. בחר בפורמט **WebP** או **MozJPEG**
4. הגדר איכות ל-**80-85%**
5. הורד את התמונות המתוקנות והחלף אותן ב-`public/images/`

#### אופציה B: שימוש ב-ImageOptim (Mac)
1. הורד מ-https://imageoptim.com/
2. גרור את כל התמונות מ-`public/images/`
3. הכלי ידחוס אותן אוטומטית

#### אופציה C: שימוש ב-TinyPNG
1. לך ל-https://tinypng.com/
2. העלה את התמונות
3. הורד והחלף

### 2. המרה ל-WebP
WebP קטן ב-25-35% מ-JPEG באותה איכות:
- השתמש ב-https://squoosh.app/ או https://cloudconvert.com/jpg-to-webp
- המר את כל התמונות ל-WebP
- עדכן את הקוד להשתמש ב-`.webp` במקום `.jpeg`

### 3. גודל תמונות מומלץ
- **Hero background**: מקסימום 1920px רוחב, 80% איכות
- **Section images**: מקסימום 1200px רוחב, 85% איכות
- **Thumbnail images**: מקסימום 800px רוחב, 85% איכות

### 4. מה כבר עשיתי
- ✅ הוספתי `loading="lazy"` לכל התמונות שמתחת ל-fold
- ✅ הוספתי `quality={85}` לכל התמונות
- ✅ הוספתי `priority` רק לתמונות קריטיות (Hero, AboutSection, GuideSection)
- ✅ הוספתי preload לתמונת ה-Hero

### 5. תוצאה צפויה
לאחר דחיסה, התמונות אמורות להיות:
- **לפני**: 300-360KB כל אחת
- **אחרי**: 80-150KB כל אחת (עם WebP)
- **שיפור**: 50-70% מהיר יותר בטעינה

## הוראות מהירות
1. העלה את כל התמונות מ-`public/images/` ל-https://squoosh.app/
2. בחר WebP, איכות 85%
3. הורד והחלף את הקבצים
4. עדכן את הקוד להשתמש ב-`.webp` במקום `.jpeg`
